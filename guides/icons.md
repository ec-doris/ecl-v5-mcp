# ECL Icons (EC preset, v5.0.1)

ECL icons are WebTools-rendered glyphs styled and sized by ECL. Their markup is
a `<span>` containing a WebTools icon class, the ECL base and size classes, and
an ECL name/family hook. No SVG icon files are bundled by this MCP package.

## Required assets

Load the EC component CSS and the WebTools loader:

```html
<link rel="stylesheet" href="assets/ecl-ec.css" />
<script src="https://webtools.europa.eu/load.js"></script>
```

In order for Icons to actually display or webtools to even load, the domain 
of the site has to be localhost or end with "*.europa.eu". Icons are used in 
many places, like breadcrumbs, mobile navigation, language menu, search, ...

If you want icons to work without being on a whitelisted domain, it is possible.
However, it is also very complicated. Most vibe coding tools should be able
to do it.

Call `guide("assets")` for the complete document setup. ECL's CSS supplies
`.ecl-icon` sizing, color, and transformations; WebTools supplies the actual
glyph. Without the WebTools loader, `wt-icon*` classes are not defined by the
local ECL assets and the icon will not render.

For reusable examples from this MCP server, call the real `component` tool with
`id: "icon"`:

```json
{
  "tool": "component",
  "parameters": { "id": "icon" }
}
```

When compiling ECL packages, the Twig component is `@ecl/icon` and the icon
name inventories are published by `@ecl/resources-icons`.

## Standard icon markup

For a standard icon named `{name}`, use both `wt-icon--{name}` and
`ecl-icon--{name}`:

```html
<span
  class="wt-icon--audio ecl-icon ecl-icon--m ecl-icon--audio"
  aria-hidden="true"
></span>
```

The `ecl-icon--audio` name hook has no standalone rule in ECL CSS, but it is the
markup generated by the official Twig component and gives components a stable
selector. Keep it with the WebTools class.

## Families and styles

Family names sit between the base and icon name:

```html
<!-- EU-member flag family -->
<span
  class="wt-icon-flags--be ecl-icon ecl-icon--m ecl-icon-flags--be"
  aria-hidden="true"
></span>

<!-- Social-network family with a WebTools visual style -->
<span
  class="wt-icon-networks--facebook wt-icon--monochrome ecl-icon ecl-icon--m ecl-icon-networks--facebook"
  aria-hidden="true"
></span>

<!-- Another WebTools family used by ECL v5 components -->
<span
  class="wt-icon-phosphor--star ecl-icon ecl-icon--m ecl-icon-phosphor--star"
  aria-hidden="true"
></span>
```

Use family `flags` for the EU member flag codes listed below and `networks` for
the network names. WebTools style classes such as `wt-icon--primary`,
`wt-icon--monochrome`, and `wt-icon--inverted` affect the WebTools rendering;
they are distinct from ECL's color modifiers.

## ECL size, color, and transform classes

### Sizes

| Class             | EC screen size | EC print size |
| ----------------- | -------------- | ------------- |
| `ecl-icon--2xs`   | 1rem           | 12pt          |
| `ecl-icon--xs`    | 1.125rem       | 13.5pt        |
| `ecl-icon--s`     | 1.25rem        | 15pt          |
| `ecl-icon--m`     | 1.5rem         | 18pt          |
| `ecl-icon--l`     | 2rem           | 24pt          |
| `ecl-icon--xl`    | 2.5rem         | 30pt          |
| `ecl-icon--2xl`   | 3rem           | 36pt          |
| `ecl-icon--fluid` | 1em            | 1em           |

The official Twig default is `m`. Use `fluid` when the icon should track its
element's font size.

### Colors

- No ECL color modifier: `.ecl-icon` uses `fill: currentcolor`.
- `ecl-icon--primary`: uses the active color mode's on-surface primary token,
  falling back to the EC primary color.
- `ecl-icon--inverted`: uses white on screen and the inverted color-mode token
  with a white fallback in print.

There is no `ecl-icon--default` CSS rule. In direct markup, omit the modifier
for the default current color. Call `guide("colours")` for color-mode classes.

### Transformations

- `ecl-icon--rotate-0`, `ecl-icon--rotate-90`, `ecl-icon--rotate-180`, and
  `ecl-icon--rotate-270`.
- `ecl-icon--flip-horizontal` and `ecl-icon--flip-vertical`.

The 90° and 270° transformations swap direction in an RTL context. The base
icon has a 300ms transform transition. Use at most one transform modifier;
separate transform rules overwrite rather than compose with one another.

## Accessibility

Icons should reinforce meaning, not replace an understandable label.

For a decorative icon or one next to visible text, hide it:

```html
<a class="ecl-link" href="report.pdf">
  <span
    class="wt-icon--download ecl-icon ecl-icon--fluid ecl-icon--download"
    aria-hidden="true"
  ></span>
  Download the report
</a>
```

For an icon-only control, name the control and keep the glyph hidden:

```html
<button
  class="ecl-button ecl-button--primary"
  type="button"
  aria-label="Search"
>
  <span
    class="wt-icon--search ecl-icon ecl-icon--m ecl-icon--search"
    aria-hidden="true"
  ></span>
</button>
```

When a standalone icon itself must be exposed as an image, the v5.0.1 Twig
component uses `aria-hidden="false"`, `role="img"`, and a `title` attribute:

```html
<span
  class="wt-icon--warning ecl-icon ecl-icon--m ecl-icon--warning"
  aria-hidden="false"
  role="img"
  title="Warning"
></span>
```

Describe its function or meaning, not its appearance, and do not include the
word “icon.” The Twig inputs are `as_image`, `icon.title` (or
`extra_accessibility.title`), and optional `extra_accessibility.description`,
which is output as `data-desc`. Supplying a title automatically selects image
semantics. Prefer visible text or an accessible name on the containing control
where possible.
